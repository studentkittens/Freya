Eine liste von features der GUI (das gehört eigentlich ins Pflichtenheft?)
---------------------------------------------------------------------------

Von links oben nach rechts unten.

Menü:
-----
Überall Anzeige der Keyshortcuts.
- File
  * connect, insens. wenn connected
  * disconnect, insens. wenn disconnected
  * quit 
- Playback 
  * next, prev, play, stop
  * consume, single, random, repeat - anzeige mit Haken falls aktiviert
- Misc
  * Inc/Descreasevolume
- Help
  * About Dialog

-------
Obere Zeile: 
 - Anzeige des Titels (In klammern anzeiges der Tracknummer)
   * Falls nicht connected wird eine entsprechende Nachricht angezeigt
 - Timeslide
   * Passt sich automatisch der aktuellen Position an
   * Ein Drag&Drop an die richtige Position sprint innerhalb des Songs,
     auch während des Ziehens (alle 0.05 Sekunden)
   * Bei Stop auf 0, oder hält bei pause
 - Stopbutton (Stoppt Lied, setzt Timeslide zurück)
 - Pausebutton (zeigt ein play icon, wenn pausiert, und andersrum)
 - Previous
 - Next
 - Volumebutton (Popupslider)

Untere Zeile
  - Anzeige von "by $(Artist) on $(Album) ($(date))
    * Falls nicht connected oder nicht spielend wird "Not Playing angezeigt"

--------

- Links ist die Sidebar die eine Liste von Browsern zeigt
  * Bei Klick auf einem Browser wird er ausgewählt
  * der Inhalt wird im Pane daneben angezeigt
  * Sollte die Browserliste überfüllt sein wird ein Scrollbalken angezeigt
- Nach einem Separator findet sich ein userinaktives widget das den Song anzeigt der als nächstes spielen wird
  * Falls kein nächster Song wird eine entsprechende Nachricht angezeigt 
- Darunter findet sich  eine box mit 4 Iconbuttons die eindrückbar sind.
  * Repeat
  * Consume
  * Repeat
  * Single
  Der eingedrückte Zustand wird automatisch geändert falls nötig.
  
----- 

Darunter findet sich die Statusbar:
- Falls connected zeigt sie: 
  * Audiorate
  * Bitzahl
  * Kbitanzahl
  * Mono/Stereo (der Server unterstützt nur die Anzeige der beiden Arten, 5.1 == Stereo)
  * Menge der Songs in der Datenbank (nicht in der Queue!)
  * Die zeit die benötigt wird um die ganze Datenbank abzuspielen
  * Lautstärke 0-100
 

 Browser
 =======

 Queue
 - Zeigt die aktuelle Warteschlange an, in den Spalten
   * Artist/Album/Title
   * Spalten sind frei anordenbar
 - Sonst besteht die Queue nur aus dieser Liste und einem Suchfeld
   * Tippt man im Suchfeld so springt man automatisch zu den Artist der mit den getippten Buchstaben anfängt.
     # 'Ak' würde zu 'Akrea' springen
   * Aktivieren des Suchfelds (entweder über enter oder dem Suchicon) wird die auswahl durch eine volltextsuche gefiltert
 - Auswahl entweder durch linksklick, kombiniert mit shift/strg oder durch Auswählen mit der Maus ("Rubberbanding")
 - Ein Rechtsklickmenü bietet die folgenden Möglichkeiten:
   * Remove: Entfernen der Songs aus der Queue
   * Clear: Entfernen aller Songs aus der Queue
   * Save as playlist: Die gesamte Queue wird als playlist abgespeicher,
     Der name der neuen playlist wird durch einen Dialog abgefragt.
 - Bei Änderungen des Server wird die Queue im Hintergrund geupdatet

 Playlistmanager
 - Zeigt alle vorhandenen auf dem Server gespeicherten Playlisten
 - Es wird der Playlistname und das letzte Änderungsdatum angezeigt
 - Ein Rechtsklickmenü bietet die folgenden Operationen:
   * Append: Fügt den Inhalt der Playliste der Queue am Ende hinzu
   * Replace: Ersetzt den Inhalt der Queue mit dieser Playlist
   * Delete: Entfernt die ausgewählten Playlisten unwiderruflich
 - Die Namenszellen sind editierbar, sodass der Playlistname einfach geändert werden kann
 - Für spätere Verwendung gibt es ein Statuslabel das Infos oder Fehlermeldungen zeigt. 

 Database
 - Der Databasebrowser zeigt eine Visualisierung der Datenbank die an einen Filebrowser angelehnt ist
   * Anzeige von Songs und Files durch unterschiedliche Icons
   * Anzeige des "Dateinamen" unter dem Icon
   * Doppelklick oder Enter auf einen Ordner bewirkt ein Absteigen in diesem
   * Doppelklick oder Enter auf ein Songfile bewirkt ein Hinzufügen dessen zur Ende der Queue
   * 'Backspace' geht ein Verzeicniss nach oben.
   * Am unteren Rand befindet sich eine Steuerleiste:
     # homebutton: Geht zum Wurzelverzeichniss zurück
     # Zurückbutton: Dasselbe wie backspace
     # Das Suchfeld filtert die momentane Anzeige
     # ganze rechts zeigt ein Label den aktuellen Pfad an
   * Ein Kontextmenü bietet folgende Optionen:
     # Add: Fügt die ausgewählte Menge der Queue hinzu, rekursiv falls sich Verzeichnisse darunter befinden
     # Add All: Fügt alles, ungeachtet der auswahl der queue hinzu (performatere Version von add)
     # Replace: Wie add, löscht aber vorher die Queue
     # Update: Weist den Server an die Datenbank zu aktualisieren, und neue/veränderte files zu aktualisieren
     # Rescan: Weist den Server die Datenbank zu aktualisieren; untersucht alle files von neuen (teuer)

 Statistics
 - Zeigt verschieden Statistiken in einfacher Label:Value Form an
 
 Settings
 - Tabbasiertes Layout
 - Falls die verbindung verloren soll zum Settingsbrowser gesprungen werden
 - Muss auch ohne verbindung zum server funktionsfähig sein
 - Networktab: 
   * Hostname und Proxy
   * Maximaler Zeit die zum reconnect benötigt wird.
   * Eine Liste von lokal im Netzwek vorhandenen soll angezeigt werden
   * Ob der Client sich automatisch connecten soll
 - Playback
   * Crossfade zwischen Songs (weicher übergang zwischen 2 Songs)
   * Stoppen der Musik beim Schließen des clients
 - General
   * Einstellungen zu libnotify und trayicon
 - Unter Outputliste soll eine Liste von Ausgabegeräten des Servers angezeigt werden
   * innerhalb der Liste soll es togglebutton geben um den output entweder zu en oder disablen
 - Alle Settingstabs ändern die Werte nicht sofort: 
   * Sie werden erst persistent übernommen wenn der user die Konfiguration abspeichert
   * Der Rückgängig button setzt die Präsentation auf die letzen persistent gesetzten Werte
   * "Zurücksetzen" lädt überall die Defaultconfig

START
=====

(Erster Start wurde ja schon abgehandelt)
- Beim Start des Client wird falls eine Verbindung besteht ein bestimmter Browser angezeigt.
  Der Client ist von Start an voll funktionsfähig
- Ohne verbindung ist die GUI gelockt bis auf das Menü gelockt, und der Settingsbrowser ist ausgewählt.
  Hier können dann (zb. beim ersten start entsprechenden Änderungen vorgenommen werden)
